# AI Video Studio — Agent Harness 设计文档

## 1. 核心设计理念

### 1.1 产品能力上限 = 设计质量，而非 LLM 能力

即梦的设计哲学是：**LLM 是执行者，不是决策者**。产品的创意质量上限由以下因素决定：

- **流程设计**：分镜表的颗粒度、素材映射机制、prompt 格式规范
- **Prompt 工程**：系统提示词的精确程度决定了 LLM 输出的质量下限
- **编排逻辑**：哪些步骤并行、哪些串行、在哪里让人介入决策

这意味着：**即使用户使用不同的 LLM（GPT-4o、DeepSeek、MiMo），只要 harness 设计得好，最终视频质量的差异可以控制在可接受范围内。**

### 1.2 从即梦学到的三个关键设计原则

| 原则 | 即梦的做法 | 我们的设计 |
|------|-----------|-----------|
| **选择优于生成** | 先给用户4个方向选择，再深入 | 每个关键节点提供选项而非直接生成 |
| **素材即引用** | `@图片1` 映射机制，prompt 中引用而非描述 | 建立素材-引用ID映射表，prompt 中使用引用 |
| **结构化约束** | 严格的prompt格式（镜头号/景别/画面/声音/运镜/时长） | 定义标准化输出 schema，强制 LLM 按格式输出 |

## 2. Agent 架构设计

### 2.1 Agent 类型定义

```
┌─────────────────────────────────────────────────────────┐
│                    Agent 类型体系                         │
│                                                         │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │ Director Agent│  │ Creator Agent│  │ Executor Agent│  │
│  │  (决策型)     │  │  (创作型)    │  │  (执行型)     │  │
│  │              │  │              │  │              │  │
│  │ - 方向选择    │  │ - 剧本创作   │  │ - 素材生成    │  │
│  │ - 风格确认    │  │ - 分镜设计   │  │ - 视频合成    │  │
│  │ - 质量评审    │  │ - Prompt编写 │  │ - TTS配音     │  │
│  │ - 修改决策    │  │ - 旁白撰写   │  │ - 字幕生成    │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
│         │                 │                 │           │
│         └─────────┬───────┘                 │           │
│                   ▼                         │           │
│           ┌──────────────┐                  │           │
│           │  Orchestrator │◄─────────────────┘           │
│           │  (编排器)      │                              │
│           │               │                              │
│           │ - 阶段流转     │                              │
│           │ - 状态管理     │                              │
│           │ - 事件总线     │                              │
│           │ - 用户确认点   │                              │
│           └──────────────┘                               │
└─────────────────────────────────────────────────────────┘
```

#### Director Agent（决策型）
- **职责**：在关键节点向用户呈现选择，而非自主决策
- **输出**：结构化的选项列表（3-4个方向）
- **触发点**：流程开始、风格选择、质量不达标时

#### Creator Agent（创作型）
- **职责**：生成创意内容（剧本、分镜、prompt）
- **输出**：严格遵循 schema 的结构化内容
- **特点**：有明确的创作约束和格式规范

#### Executor Agent（执行型）
- **职责**：调用外部工具完成具体任务
- **输出**：文件路径、任务状态
- **特点**：确定性执行，不涉及创意决策

### 2.2 编排器（Orchestrator）设计

即梦使用了**技能激活（Skill Activation）**模式：每个阶段是一个独立技能，编排器按序激活。

```python
# 我们的设计：阶段式流水线 + 用户确认点
class PipelineStage(Enum):
    DIRECTION = "direction"         # 创意方向选择
    SCRIPT = "script"               # 剧本创作
    STYLE_EXPLORATION = "style"     # 风格探索
    MATERIAL_GENERATION = "material" # 素材生成
    STORYBOARD = "storyboard"       # 分镜设计
    PROMPT_GENERATION = "prompt"    # Prompt 生成
    VIDEO_GENERATION = "video"      # 视频生成
    ASSEMBLY = "assembly"           # 合成输出

class PipelineOrchestrator:
    """编排器：管理阶段流转和用户确认点"""

    # 每个阶段之间的确认点
    CONFIRMATION_POINTS = {
        PipelineStage.DIRECTION: "请选择创意方向",
        PipelineStage.SCRIPT: "请确认剧本内容",
        PipelineStage.STYLE_EXPLORATION: "请选择视觉风格",
        PipelineStage.MATERIAL_GENERATION: "请确认参考素材",
        PipelineStage.STORYBOARD: "请确认分镜表",
        PipelineStage.PROMPT_GENERATION: "请确认视频Prompt",
    }

    async def advance(self, user_input: str = None):
        """推进到下一阶段"""
        current = self.state.current_stage

        # 如果当前阶段有确认点，需要用户确认
        if current in self.CONFIRMATION_POINTS and not self.state.confirmed:
            await self.request_confirmation(self.CONFIRMATION_POINTS[current])
            return

        # 执行当前阶段
        result = await self.execute_stage(current)

        # 如果需要用户确认，暂停
        if self.needs_confirmation(current):
            self.state.confirmed = False
            await self.present_result(result)
        else:
            # 自动进入下一阶段
            self.state.current_stage = self.next_stage(current)
```

## 3. Prompt 设计模式

### 3.1 即梦的 Prompt 设计分析

从聊天记录中提取的关键 prompt 模式：

#### 模式一：选择式 Prompt（Director Agent）

```
我来帮你构思几个不同方向的末世囤货视频创意:

1. 普通人日常风
   讲述一个平凡打工人在末世爆发前突然感知到危机...

2. 反差感幽默风
   一个看似柔弱的女生/上班族，其实暗地里是囤货大佬...

3. 丧尸爆发紧急型
   病毒突然扩散，城市陷入混乱，主角在封锁前最后几小时疯狂采购...

4. 废土生存长远型
   世界已经末世，主角驾驶改装越野车在废弃城市里搜寻物资...

你更想做哪个方向？
```

**设计要点**：
- 每个选项有**明确的风格标签**（日常风/幽默风/紧急型/长远型）
- 每个选项有**具体的场景描述**（不是抽象概念）
- 选项之间有**明显的差异性**（不是微调变体）

#### 模式二：结构化剧本 Prompt（Creator Agent）

```
根据story-script的要求来创作:
1. 先列出角色清单
2. 分场景创作，每个场景包含：地点、时间、内/外
3. 动作和对话要符合可拍摄原则
4. 最后写入文件并展示给用户
```

**设计要点**：
- 明确的**输出结构**（角色清单 → 场景 → 动作/对话）
- 明确的**约束条件**（可拍摄原则）
- 明确的**输出格式**（写入文件）

#### 模式三：素材映射 Prompt（关键创新）

```
@图片1 = 公寓客厅清晨 (apartment_livingroom_realistic_film)
@图片2 = 陈峰 (chenfeng_character_main)
@图片3 = 智能手机 (smartphone_item_main)
@图片4 = 大容量拉杆箱 (suitcase_item_empty)

【场面调度】公寓客厅清晨场景中@[图片1]，陈峰@[图片2] 在画面中景活动，
智能手机@[图片3] 放置在茶几上，大容量拉杆箱@[图片4] 空箱状态...
```

**设计要点**：
- 建立**素材-ID映射表**（人可读名称 + 机器ID）
- prompt 中使用**引用而非描述**（`@[图片1]` 而非 "一个公寓客厅"）
- 确保生成的视频**角色/场景一致性**

#### 模式四：镜头级 Prompt（精细控制）

```
「镜头1-1：（特写，平视）黑色智能手机屏幕亮起，弹出红色紧急推送文字，
"未知病毒扩散，本市即将封城"，红色字体闪烁，固定。」手机消息提示音
```

**设计要点**：
- **镜头编号**：场景-镜头 二级编号（1-1, 1-2, ...）
- **景别角度**：特写/近景/中景/全景/远景 × 平视/俯视/仰视
- **运镜方式**：固定/移镜/跟镜/推镜/拉镜
- **声音设计**：每个镜头独立的音效标注
- **时长控制**：每个镜头有精确的秒数

## 5. 与即梦的关键差异

### 5.1 即梦的优势（我们学习的）

| 特点 | 即梦做法 | 我们的实现 |
|------|---------|-----------|
| 技能系统 | 内置多种技能（video-sop, story-script, script-chunk） | Python sidecar 中实现独立 service |
| 素材映射 | `@图片1` 引用机制 | 同样实现引用映射表 |
| 并行生成 | 4 种风格同时生成 | asyncio 并行调用 |
| 渐进式确认 | 每个阶段都有确认点 | Orchestrator 确认点机制 |

### 5.2 我们的优势（超越即梦的）

| 特点 | 即梦限制 | 我们的设计 |
|------|---------|-----------|
| **素材来源** | 仅 AIGC 生成 | 素材拼接 + AIGC 双模式 |
| **LLM 灵活性** | 绑定特定 LLM | 支持 20+ LLM 供应商 |
| **TTS 选择** | 绑定特定 TTS | 支持 Edge TTS / MiMo / ElevenLabs |
| **离线能力** | 需要联网 | 本地 FFmpeg + 本地 Whisper |
| **开源可控** | 闭源 | 完全开源可控 |

### 5.3 我们需要额外设计的

1. **素材拼接模式的分镜适配**：即梦的分镜是为 AIGC 设计的，我们需要适配"从素材库搜索匹配"的模式
2. **质量回退机制**：当 LLM 输出不符合 schema 时的重试和降级策略
3. **成本控制**：AIGC 生成成本高，需要智能判断哪些镜头用 AIGC、哪些用素材
