# AI 视频创作工作台 — 产品逻辑文档

## 一、核心理念

> 文生视频大模型已进入音视频一体化时代。豆包等主流供应商的最新模型默认生成带音频的视频。
> 我们的产品直接利用这一能力，不需要单独的 TTS 合成步骤。

**设计原则：LLM 是执行者，不是决策者。每一步用户都有确认权。**

---

## 二、五阶段流程

```
┌─────────────────────────────────────────┐
│  Stage 1: AI创作（Agent 编排）            │
│                                         │
│  ① 用户输入主题                          │
│  ② Director Agent → 4 个创意方向          │
│     用户选择一个方向                      │
│  ③ ScriptWriter Agent → 结构化剧本        │
│     标题、角色、场景、旁白、对白           │
│     用户确认/编辑剧本                    │
│  ④ Storyboard Agent → 分镜表             │
│     镜号、景别、画面描述、旁白段落、时长    │
│     用户确认分镜                         │
│                                         │
│  输出：分镜表 JSON                       │
│  每个镜头 = {                            │
│    shot_id, shot_type, angle,           │
│    description(画面描述),                │
│    narration(旁白文本),                  │
│    duration(时长)                        │
│  }                                      │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  Stage 2: 文案微调（可选）                │
│                                         │
│  - 显示完整旁白文本                      │
│  - 用户可修改措辞、调整节奏              │
│  - 修改后的旁白自动同步回分镜表           │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  Stage 3: 逐镜头生成（AIGC 音视频一体）   │
│                                         │
│  对分镜表每个镜头：                      │
│    AIGC(画面描述 + 旁白文本)              │
│    → 带音频的视频片段                    │
│                                         │
│  用户可：                                │
│    - 预览每个镜头                        │
│    - 对单个镜头重新生成                  │
│    - 切换 AIGC 工具                     │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  Stage 4: 字幕 + BGM                    │
│                                         │
│  - 字幕样式配置（字体、颜色、位置）        │
│  - BGM 选择（无 / 随机 / 自定义）         │
│  - BGM 音量调节                         │
└──────────────┬──────────────────────────┘
               │
               ▼
┌─────────────────────────────────────────┐
│  Stage 5: 总合成 + 预览                  │
│                                         │
│  - FFmpeg 拼接所有镜头片段               │
│  - 叠加字幕（防止 AIGC 文字乱码）         │
│  - 混合 BGM                            │
│  - 右侧预览最终视频                      │
│  - 导出 .mp4                           │
└─────────────────────────────────────────┘
```

## 三、四个标签页的职责

| 标签页 | 职责 | 数据流向 |
|--------|------|---------|
| **AI创作** | Stage 1: Agent 编排，产出结构化剧本 + 分镜表 | → 共享 state |
| **文案** | Stage 2: 微调旁白文本（可选） | ↔ 共享 state |
| **AIGC** | Stage 3: 逐镜头生成带音频的视频 | ← 读取分镜表 |
| **字幕** | Stage 4: 字幕样式 + BGM 配置 | → 传递给合成 |

右侧 **视频预览** 始终可见，负责 Stage 5 的合成触发和结果展示。

## 四、数据流

```
AI创作标签
  │
  ├─ directions: [{id, label, concept, description}]
  ├─ script: {title, characters, scenes}
  │           scenes[].narration = 该场景的旁白文本
  └─ storyboard: {shots: [{shot_id, shot_type, angle,
                            description, narration, duration}]}
                          │
                          ▼
              WorkspaceLayout 共享 state
              ├─ subject: string
              ├─ shots: Shot[] (分镜数据，含画面描述+旁白)
              ├─ subtitleConfig: {font, position, color, ...}
              └─ bgmConfig: {type, volume, path}
                          │
                          ▼
              VideoPreview → Pipeline
              │
              ├─ 对每个 shot:
              │    AIGC(description + narration) → 带音频的视频片段
              ├─ 拼接所有片段 → combined.mp4
              ├─ 叠加字幕 → with_subtitles.mp4
              ├─ 混合 BGM → final.mp4
              └─ 预览 + 导出
```

## 五、与即梦的对比

| 维度 | 即梦 | 我们 |
|------|------|------|
| 交互方式 | 纯对话 | 可视化面板 + 确认按钮 |
| 剧本格式 | .md 文档 | 结构化 JSON（可导出 .md） |
| 分镜控制 | 自动生成 | 用户可逐镜头编辑/替换 |
| AIGC 工具 | 仅即梦 | 多工具可切换（豆包/可灵/Runway/Pika/Luma） |
| 音频 | 工具自带 | 工具自带（音视频一体化） |
| 字幕 | 人工后期 | 自动生成 + 样式可配 |
| 适用场景 | 在线单次创作 | 桌面端批量/离线创作 |
